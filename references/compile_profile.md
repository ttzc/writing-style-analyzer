# 全量样本重编译画像

当用户积累了多篇代表性样本后，对所有归档样本进行一次全量重分析，生成更精准的写作习惯画像。

触发词："重新整理画像"、"全量分析"、"用所有样本更新我的画像"、"compile profile"、"rebuild profile"。

## 使用场景

- 已积累 2+ 个归档样本，想交叉验证模式（建议 3+ 以获得更可靠结果）
- 增量分析的画像出现多处冲突标注，需要全量重新裁决
- 觉得当前画像不够准，想从头重建

## 前置检查

先检查 `data/samples/INDEX.md` 是否存在且有至少 2 条记录。不满足则提示：

> "归档的样本还比较少（需要至少 2 篇代表性样本），建议先多分析几篇再全量整理。"

## 工作流

### Step 1: 读取所有样本

遍历 `data/samples/INDEX.md` 中的每条记录，读取对应的 `aigc_original.md` 和 `user_modified.md`。

### Step 2: 逐对分析

对每对样本，直接阅读 `aigc_original.md` 和 `user_modified.md`，进行五大维度分析（同 `references/analyze_single.md` 的 Step 1）。LLM 直接对比，无需任何脚本。

### Step 3: 跨样本交叉验证

所有样本分析完成后，对每个维度逐条交叉验证：

```
对于维度 X 中的模式 P（总样本数 = N）：
  ├── ≥4 次绝对计数 且 占比 ≥ 80% → 标注"稳定"
  ├── 2-3 次绝对计数 或（N ≥ 5 时占比 40-79%）→ 标注"多次观察"
  ├── 仅 1 次观察 或（N ≥ 5 时占比 < 40%）→ 标注"初步"
  └── 样本间方向矛盾 → 标注"待裁决"，列出矛盾样本
```

**矛盾裁决规则**：
- 如果某个模式在一个样本中出现，但在另一个中呈现相反方向（如：样本 A 显示用户偏好短句，样本 B 显示用户偏好扩展句子），记录两种观察并分析上下文差异（是否因为文章类型不同？技术文档 vs 随笔风格有别？）
- 如果能识别出"分场景"的合理原因，拆分为两个场景画像（如"技术文档：短句偏好" / "随笔：中等句长"）

### Step 4: 生成全量画像

用 `assets/profile_template.md` 为骨架，按交叉验证结果填充所有维度。

画像头部更新（保留 `sample_count` 和 `last_updated` 以兼容增量分析）：
```markdown
> 基于 {sample_count} 篇代表性样本全量分析 | 最后更新：{last_updated}
> 其中稳定模式 {X} 条，多次观察 {Y} 条，初步 {Z} 条，待裁决 {W} 条
```
- 若画像超过约 3000 字符，将"稳定"层级的旧模式压缩为简洁陈述，保持总量可控

### Step 5: 写入 + 输出摘要

覆盖 `data/profile.md`。写入后自动 git commit（若 git 可用）：
```bash
git -C data add profile.md && git -C data commit -m "profile: 全量重编译 (N篇样本)"
```

输出：
- 总样本数、稳定/多次观察/初步/待裁决 各多少条
- 3-5 个最有代表性的习惯（标注为"稳定"且覆盖多维度）
- 待裁决的冲突（如有）
- 分场景画像（如有）

## 与增量分析的差异

| | 增量分析 | 全量重编译 |
|------|------|------|
| 输入 | 1 对样本 | 所有归档样本 |
| 画像更新 | 合并追加 | 覆盖重写 |
| 可信度 | 依赖历史标注 | 本次交叉验证重新判定 |
| 冲突处理 | 标注 + 等待 | 本次裁决 |
| 适用时机 | 每次分析后 | 积累多篇后集中整理 |
