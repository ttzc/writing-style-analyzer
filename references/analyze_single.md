# 单篇文章分析工作流

对比一对 AIGC 原文与用户修改版，提取该样本中的写作习惯模式，累积更新画像。

## Input Detection

收到请求后先判断输入形式：

### 文件路径
两个绝对路径 → 用 Read 读取。

### 粘贴文本
文本中包含明确的分隔标记：

```
【AIGC 原文】
<AI 生成的文章内容>

【用户修改版】
<用户修改后的文章内容>
```

直接解析两段。

### 单文件 + 上下文
只给一个文件路径，检查文件内是否包含两个版本的分隔标记。无法确定则追问另一个版本。

---

## Step 1: 对比分析（LLM 直接分析，无脚本）

同时阅读 AIGC 原文和用户修改版，直接进行对比分析。可选先跑 `diff -u aigc.md user.md` 获取快速概览，但核心分析由 LLM 完成。

加载 `references/analysis_dimensions.md`，对五大维度逐一分析：

1. **结构性** — 段落/句子长度、标题层级、列表偏好、结构重组
2. **词汇** — 用词替换模式、专业术语偏好、口语/书面语倾向
3. **句式** — 语态、连接词、句复杂度、句首多样性
4. **语气与风格** — 正式程度、情感表达、人称、标点
5. **内容操作** — 添加什么、删除什么、改写模式分类

每个维度输出：
- 观察到的模式（具体实例，直接引用原文中的改动）
- 量化观察（如"平均句长从 X 降到 Y"、"删除了 N 处冗余修饰"）
- 可信度（初步/多次观察/稳定/待裁决）

---

## Step 2: 合并现有画像

若 `data/profile.md` 存在，逐条对比：

| 新旧关系 | 处理 |
|----------|------|
| 一致 | 增强可信度，"初步"→"稳定"，追加样本计数 |
| 矛盾 | 标注冲突点，记录双方观察，等待后续样本裁决 |
| 全新 | 追加条目，标注"初步" |

更新 `last_updated` 和 `sample_count`。在 `change_log` 追加摘要。

画像不存在则用 `assets/profile_template.md` 初始化。

---

## Step 3: 写入画像

写入 `data/profile.md`。超过约 3000 字符时，将已确认"稳定"的旧模式压缩为简洁陈述。

写入后自动 git commit（若 git 可用）：
```bash
git -C data add profile.md && git -C data commit -m "profile: 增量更新 #N"
```

---

## Step 4: 输出摘要

不输出完整画像，只输出：
- 样本编号（第几次分析）
- 3-5 个关键新发现
- 与之前画像一致/矛盾的发现
- 画像存储位置

---

## Step 5: 询问是否归档

> "这次对比有代表性吗？如果值得归档，我会把 AIGC 原文和你的修改版都保存下来。"

确认后：

1. 确定序号：读取 `data/samples/INDEX.md` 中当天已有的归档数，序号 = 已有数 + 1（两位补零，如 01, 02）
2. 创建 `data/samples/{YYYY-MM-DD}-{NN}/`
3. 保存 `aigc_original.md`、`user_modified.md`、`diff_summary.md`
4. 更新 `data/samples/INDEX.md`：
   ```
   | 日期 | 序号 | 标题 | 关键词 |
   | 2026-07-11 | 01 | <标题> | <3个关键词> |
   ```
5. git commit（若可用）：
   ```bash
   git -C data add -A && git -C data commit -m "archive: YYYY-MM-DD-NN <标题>"
   ```

内部归档判断标准（不逐条念给用户）：
- 修改幅度明显（用户做了实质性改写，非仅修正错别字）
- 至少 3 个维度有显著差异
- 用户主动修改（非随机编辑）
- 典型写作场景
